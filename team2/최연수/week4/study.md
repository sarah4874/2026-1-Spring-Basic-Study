### **섹션5**

###### **프로젝트 생성**

* 프로젝트: 빌드도구, Gradle이 가장 발전됨, 언어는 자바 선택, 
* 메타데이터: 프로젝트는 밖에서 봤을때를 나타냄, 이름이나 설명 Jar은 자바 안에 있는 모든걸 압출, War은 웹 안에있는 것들을 압축
* Dependencies: 사용할 것들 고르기(AI, Spring web 등)
* 프로젝트 생성 이후, 인텔리제이에서  build.gradle 잡고 열기  

###### **폴더 설명**

* application.propertires 파일을 통해서 DB를 가져올 수 있음.
* build.gradle: 빌드할 때 사용할 문서(중요함)/간결하게 언어를 쓸 수 있게 되어있음/dependencies블럭을 통해서 추가되어있는 것들을 확인할 수 있음
* gradlew: 자료 파일을 만들 수 있음.
* settings.gradle: 세팅에 관련되어 설정할 것들을 적음.
* extternal libraries: 미리 만들어 놓은 코드들 확인 가능



### **섹션6**

###### **자바로 개발하면서 불편한 점**

* 객체 지향 언어: 객체(메소드 호출, 로직 짜기..)+로직->객체에 대해 늘 고려해야됨

###### **스프링의 주요 원리**

1. IoC(Inversion of Control/ 제어의 역전): 객체의 흐름(프로그램의 흐름)에 대한 제어권을 정반대로 뒤집는다.(제어권이 개발자에서 스프링이 가지게됨)
2. 컨테이너: 객체를 생성해서 관리할 때 사용하는 박스
3. 스프링빈: 스프링이 컨테이너에 담아두고 관리하는 객체
4. DI(Dependency Injection/의존성 주입): 개발자는 주입 받고, 스프링은 주입해줌(스프링이 제어권을 가져갔기 때문)

\+객체를 사용한다: 객체를 생성해서 메소드/필드를 호출/접근한다.

+"IoC를 구현하기 위해 DI가 필요하다"
